Fork-Join in OneAPI TBB

OneAPI Thread Building Block (TBB), aka intel TBB, provides a work-stealing fork-join scheduler. This is to compare with previous blog post on Javaâ€™s ForkJoinPool.

GitHub: https://github.com/ragas/SyncSillies/CppFJ

Intro

In the previous blog post on Javaâ€™s ForkJoinPool we created an example in Java for matrix multiplication using the default Fork-Join Pool executor service, now we do exactly same using OneAPI TBB in C++ (oneapi tbb github).

Installation

This part is quite easy if using CMake build. Simply add onetbb github repo using FetchContent.

include(FetchContent)
FetchContent_Declare(
  onetbb
  GIT_REPOSITORY https://github.com/oneapi-src/oneTBB
  GIT_TAG        v2021.12.0
)

FetchContent_MakeAvailable(onetbb)

Using TBB

The core multiplication is kept same. As refresher, we uses the divide-and-conquer algorithm to multiply two matrices. And matrices are kept as power of 2. Each sub-task is submitted using a lambada.

The parallelism is set using global_limit.


oneapi::tbb::global_control global_limit(
          oneapi::tbb::global_control::max_allowed_parallelism, parallelism);

We divide each parallel_multiply into 8 sub-tasks which we combine to form the matrix. To submit all the tasks we use oneapi::tbb::parallel_invoke, which allows a series of lambdaâ€™s to be sent for execution.

    oneapi::tbb::parallel_invoke(

        [&] {
          A1 = parallel_multiply(A, B, rowA, colA, rowB, colB, newSize,
                                 work_size);
        }, //... etc
